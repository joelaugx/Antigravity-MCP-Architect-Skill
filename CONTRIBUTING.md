# Contributing to MCP Architect

Thank you for your interest in contributing to MCP Architect! This skill was created to help the community overcome the challenges of creating and managing MCP servers.

## How to Contribute

### 1. Share Your MCP Experiences

Have you encountered an MCP issue not covered in the skill? Please share:

- **Issue description**: What went wrong?
- **Error messages**: Exact error output
- **Solution**: How did you fix it?
- **Configuration**: Sanitized config examples

### 2. Add New Examples

We welcome real-world MCP examples:

- **Service integrations**: Slack, Notion, Trello, etc.
- **Authentication patterns**: New OAuth flows, API key patterns
- **Deployment scenarios**: Docker, serverless, cloud platforms

### 3. Improve Documentation

Help make the skill more accessible:

- **Clarify instructions**: Make steps easier to follow
- **Add diagrams**: Visual explanations of MCP architecture
- **Translate**: Help non-English speakers
- **Fix typos**: Every improvement counts

### 4. Enhance Diagnostic Tools

Improve the diagnostic script:

- **New checks**: Additional validation logic
- **Better error messages**: More helpful output
- **Performance**: Faster execution
- **Cross-platform**: Windows, Linux support

## Submission Guidelines

### For Issues

```markdown
**MCP Name**: [e.g., Google Drive]
**Error Type**: [e.g., 404, OAuth, Import Error]
**Error Message**: 
```

[paste error here]

```

**Environment**:
- OS: [macOS, Linux, Windows]
- Python/Node version: [e.g., Python 3.11, Node 20]
- MCP type: [Python, Node.js, Docker]

**What you tried**: [steps taken]
**Solution**: [if found]
```

### For Examples

1. **Create a new file** in `examples/` directory
2. **Include**:
   - Complete configuration
   - Setup instructions
   - Authentication steps
   - Testing commands
3. **Sanitize**: Remove all credentials and personal info
4. **Test**: Verify the example works

### For Code

1. **Follow existing patterns**: Match the style of current code
2. **Comment your code**: Explain non-obvious logic
3. **Test thoroughly**: Ensure it works on your system
4. **Document changes**: Update README if needed

## Code of Conduct

- **Be respectful**: Everyone is learning
- **Be helpful**: Share knowledge generously
- **Be patient**: Not everyone has the same experience level
- **Be constructive**: Criticism should be actionable

## Questions?

Open an issue with the `question` label or start a discussion.

## Recognition

All contributors will be acknowledged in the README. Thank you for helping make MCP development more accessible!
